When would you want to use a remote repository rather than keeping all your work
local?

   A remote repository is ideal for projects that require collaboration.  It can also be helpful when work is beign done from different computers.

Why might you want to always pull changes manually rather than having Git
automatically stay up-to-date with your remote repository?

    You might have some changes that you are not ready to share with others/the public. There might also have been changes made to GitHub that could be undone by changes pulled from a remote.
